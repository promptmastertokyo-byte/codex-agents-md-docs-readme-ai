# Blog Style Guide

`blog/drafts/` 配下の記事を書く・レビューする・改善するための基準です。

## Core Loop

```text
Write
Review
Publish
Measure
Improve
```

## Article Requirements

- タイトルに年、数字、検索語、権威性を入れる
- 冒頭150文字以内に結論を書く
- 表、具体例、FAQを入れる
- `❌` 付きで対象外・失敗例を書く
- 公式情報、確認日、監修メモを入れる
- まとめは3点までにする

## Tone

- です/ます調
- 専門的だが、読んでいて疲れない
- 「まずここを見ます」「迷ったらこうします」のように実務寄りに書く

## Markers

- `📌` 要点
- `⚠️` 注意
- `❌` 対象外・失敗
- `✅` 実行・対象例
