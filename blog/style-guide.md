# Blog Style Guide

過去の高トラフィック記事と、今回のサンプル記事レビューから整理したブログ執筆ルールです。`blog/drafts/` 配下の新規記事作成・レビュー・改善に使います。

## 目的

読者が記事を読んだあと、迷っていた判断を1つ前に進められる状態を目指します。

良い記事は、次の3つを満たします。

- 検索した疑問にすぐ答えている
- 具体例・金額・手順があり、実務で使える
- 専門的でも、読んでいて疲れない

## 読者設定

書き始める前に、次を1行ずつ決めます。

- 読者は誰か
- 何に困って検索しているか
- 読後に何を判断・実行できればよいか
- いくら、何年、何日、何％などの具体条件は何か
- 公式情報・専門家監修・実務経験のどれで信頼性を出すか

📌 読者を広げすぎないこと。会社員向け、フリーランス向け、経営者向けは、必要なら別記事に分けます。

## トーン

- です/ます調
- 専門家が同僚に説明するような、やさしい実務口調
- 教科書調よりも「まずここを見ます」「迷ったらこうします」を優先
- 冒頭に短い共感や悩みを入れる
- 結論を早めに出す

避ける表現:

- `◯◯について解説します`
- `知っておくべきポイントを紹介します`
- `場合によります` だけで終わる説明
- 専門用語を説明なしで連発する文章

言い換え例:

| 硬い表現 | 読みやすい表現 |
| --- | --- |
| 給与課税される可能性があります | 給料として税金がかかる可能性があります |
| 合理的に按分します | 業務で使った分だけに分けます |
| 当該費用 | その費用 |
| 精算対象外となります | 会社に出しても認められにくいです |

## SEOタイトル

タイトルには、次の4要素を入れます。

- 年: `2026年版`、`令和8年` など
- 数字: 金額、日数、割合、期限、回数
- 検索語: 読者が実際に入力しそうな言葉
- 権威性: `国税庁FAQベース`、`専門家監修`、`公式情報ベース` など

基本形:

```text
2026年版｜[検索語]はいくら？[具体金額]の判断基準を[権威性]で解説
```

良い例:

- `2026年版｜在宅勤務の通信費はいくら非課税？月5,000円手当との違いを国税庁FAQベースで解説`
- `2026年版｜副業20万円以下でも確定申告は必要？会社員向けに税理士監修で解説`

弱い例:

- `在宅勤務費用について解説`
- `副業と税金の注意点`
- `知っておきたい経費の基礎知識`

## 構成テンプレート

1. SEOタイトル
2. 目次
3. 導入: 悩み・共感 → 結論
4. 基本ルール: 比較表
5. 具体的な計算・手順
6. 読者タイプ別の判断ポイント
7. ❌ 対象外・失敗しやすいケース
8. 具体例: 金額や日数入り
9. 保存すべき証拠・記録
10. 関連ツール・サービスの短い紹介
11. FAQ
12. 参考資料・監修メモ
13. まとめ

## 冒頭の型

冒頭は長くしません。読者の悩みに触れて、すぐ結論を出します。

```markdown
在宅勤務が増えると、「自宅のWi-Fi代や電気代って、会社に出してもらえるのかな」と気になりますよね。

結論からいうと、在宅勤務に通常必要な通信費・電気代は、業務で使った分を計算して実費精算する形なら、給料として税金がかかりにくいです。
```

📌 最初の150文字以内に、記事の答えを入れます。

## 見出しルール

見出しは、読者の検索語や疑問に近づけます。

| 目的 | 良い見出し | 弱い見出し |
| --- | --- | --- |
| 結論 | `結論：実費精算なら非課税になりやすい` | `はじめに` |
| 基本 | `在宅勤務手当と実費精算の違い` | `基本情報` |
| 金額 | `通信費はいくらまで精算できる？` | `通信費について` |
| 手順 | `会社員が在宅勤務費用を申請する手順` | `申請方法` |
| NG | `❌ 非課税にならない在宅勤務費用` | `注意点` |
| 証拠 | `保存しておくべき証拠・記録` | `必要書類` |

⚠️ `注意点` だけの見出しは弱いです。何に注意するのかまで書きます。

## マーカー

記事を流し読みしやすくするため、次のマーカーを使います。

- `📌` 覚えておくべき要点
- `⚠️` 条件・例外・注意点
- `❌` 対象外・失敗例・誤解
- `✅` 実行すべきこと・対象になる例

使いすぎると読みにくくなるため、1見出しにつき1〜3個を目安にします。

## 表の使い方

次の内容は文章だけで説明せず、表にします。

- 手当と実費精算の違い
- 対象になるケース・ならないケース
- 金額別の判断
- 申請手順
- 保存すべき書類

表には、なるべく数字を入れます。

```markdown
| ケース | 金額 | 判断の目安 | 保存する記録 |
| --- | ---: | --- | --- |
| 毎月の在宅勤務手当 | 5,000円 | 渡し切りなら課税されやすい | 支給規程、給与明細 |
```

## 具体例

抽象的な例ではなく、実際に計算できる数字を入れます。

良い例:

```text
4,800円 × 15日 ÷ 30日 × 1/2 = 1,200円
```

弱い例:

```text
通信費の一部を業務使用分として計算します。
```

📌 読者が自分の数字に置き換えられる例にします。

## 実務レベルの基準

実務で使える記事には、次が必要です。

- 読者がそのまま使える手順がある
- 金額・日数・割合などの具体例がある
- 保存すべき証拠が書かれている
- 判断が分かれる条件が説明されている
- 公式情報や監修情報がある
- 「これは対象外」という線引きがある

⚠️ 制度・税務・法律・医療・金融の記事は、公開前に最新情報を確認します。記事内にも確認日を入れます。

## 読みやすさの基準

人気ブログ寄りにする場合は、専門性だけでなく読み心地も調整します。

- 1文は長くしすぎない
- 1段落は1〜3文にする
- 専門用語の直後にやさしい説明を入れる
- 表・箇条書き・計算例で視線を止める
- 読者の本音を短く入れる

読者の本音の例:

- `会社に聞く前に、ざっくり基準だけ知りたいですよね。`
- `全額出していいのか、一部だけなのかが一番迷うところです。`
- `領収書はあるけれど、何を添えればいいのか分かりにくいですよね。`

## 権威性・監修

権威性は、曖昧に書かず具体化します。

| 書き方 | 強さ |
| --- | --- |
| 公式情報ベース | 普通 |
| 国税庁FAQベース | 強い |
| 税理士監修 | 強い |
| 社労士監修 | 強い |
| 実務経験者が解説 | 中程度 |

監修者がいる場合は、本文末尾に次を入れます。

- 監修者名
- 資格
- 所属または肩書き
- 監修日

監修者がいない場合は、次のように書きます。

```markdown
この記事は、2026年7月10日時点で確認した公式情報をもとにした一般的な解説です。個別判断が必要な場合は、税理士・社会保険労務士などの専門家に確認してください。
```

## 関連ツール・サービス紹介

商品やサービスを入れる場合は、1見出しだけにします。

良い形:

```markdown
## 関連ツールの使いどころ

領収書や明細が散らばる場合は、会計ソフトやクラウドストレージを使って、月ごとにまとめておくと楽です。
```

避ける形:

- 本文中で何度も商品名を出す
- 判断そのものをツールで解決できるように見せる
- 読者の不安をあおって購入へ誘導する

## FAQ

FAQは3〜5問にします。本文で拾いきれない検索語を入れます。

良いFAQ:

- `在宅勤務手当が毎月5,000円なら必ず課税ですか？`
- `通信費は半分までなら非課税にできますか？`
- `電気代も通信費と同じ計算でよいですか？`
- `Wi-Fiルーターやスマホ本体代も対象ですか？`

弱いFAQ:

- `在宅勤務とは何ですか？`
- `注意点はありますか？`
- `どうすればいいですか？`

## まとめ

まとめは3点までにします。

```markdown
## まとめ

- 在宅勤務費用は、渡し切り手当ではなく実費精算にすると非課税で整理しやすくなります。
- 通信費・電気代は、業務使用分を日数・時間・床面積などで合理的に計算します。
- 領収書、明細、在宅勤務日数、計算式を残しておくと、申請や説明がスムーズです。
```

## レビュー観点

レビュー時は、次を確認します。

| 観点 | OK条件 |
| --- | --- |
| SEO | タイトルに年・数字・検索語・権威性がある |
| 導入 | 150文字以内に結論がある |
| 実務性 | 手順・計算例・保存書類がある |
| 読みやすさ | 表、箇条書き、短い段落が使われている |
| 信頼性 | 公式情報、監修、確認日がある |
| NG例 | `❌` 付きで対象外ケースが明記されている |
| FAQ | 3〜5問で具体的な疑問に答えている |

## 公開前チェック

- タイトルに年・数字・権威性が入っている
- 冒頭で結論を言っている
- 読者を広げすぎていない
- 表が2つ以上ある
- 具体的な金額・日数・割合がある
- `❌` のNGセクションがある
- 保存すべき証拠がある
- 参考資料URLがある
- 確認日または監修日がある
- まとめが3点以内になっている
