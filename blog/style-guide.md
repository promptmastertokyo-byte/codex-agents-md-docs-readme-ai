# Blog Style Guide

Reference style, distilled from past high-traffic articles on the target
WordPress site. Use this when drafting new posts under `blog/drafts/`.

## Tone

- です/ます調. Friendly but expert -- like a specialist explaining things
  to a peer, not a textbook.
- Open with a short personal anecdote or relatable frustration before
  stating the topic.
- State the conclusion early (PREP: point first, then explanation).

## Structure Template

1. **SEO title** -- include the year, a concrete number (price, amount),
   and an authority marker (e.g. "supervised by a licensed expert").
2. **Table of contents** -- list of headings.
3. **Intro** -- relatable problem -> the conclusion, stated up front.
4. **Background/basics** -- a comparison table if there is a threshold or
   rule involved.
5. **Case-by-case breakdown** -- bullet points grouped by reader type
   (job, situation, etc.).
6. **What does NOT qualify / common mistakes** -- marked with an explicit
   negative marker (see Markers below).
7. **Worked examples** -- concrete numbers, not abstractions.
8. **Evidence/record-keeping checklist** -- what to save, why.
9. **Soft product mention** (if relevant) -- one short section, not
   pushy.
10. **FAQ** -- 3-5 short Q&A pairs covering edge cases.
11. **Summary** -- 3 bullet points max, restate the core benefit.

## Markers

Use these consistently to make the post skimmable:

- `📌` a key checkpoint worth remembering
- `⚠️` an important caveat or condition
- `❌` something that does NOT qualify / a mistake
- `✅` an action item or confirmed case

## Length

Long-form and thorough -- comprehensive enough to answer the FAQ
questions before a reader has to ask them. Favor tables and bullets over
long paragraphs.

## SEO Notes

- Titles carry the year, a specific number, and an authority signal.
- Headings should map to real search queries (job titles, specific
  amounts, specific actions).
